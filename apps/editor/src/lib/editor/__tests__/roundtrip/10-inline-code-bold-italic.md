## Inline Formatting

**Bold text** appears in documentation headings and key terms.

*Italic text* is used for emphasis and technical term introductions.

***Bold and italic*** can be combined for maximum emphasis.

Inline `code` is used for file paths, variable names, and short code snippets.

A sentence with multiple inline formats: **The `vedox dev` command** starts the *local development server* on `127.0.0.1:3001`.

**Bold with `inline code` inside** it.

*Italic with `inline code` inside* it.

A file path: `apps/editor/src/lib/editor/Editor.svelte`

A function call: `serializeDocument(frontmatter, body)`

An error code: `VDX-404`

Mixing all three: The **`DocStore`** interface defines *atomic* write semantics: `write → fsync → rename`.
